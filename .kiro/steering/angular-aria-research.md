# Angular Aria Research Project

## Project Context

This is a research-phase project focused on learning and prototyping with Angular Aria. The goal is to explore accessibility features in Angular through hands-on experimentation before formal spec-driven development.

## Research Methodology

1. **Prototype First** - Build small, focused examples to understand library behavior
2. **Learn on Fly** - Discover Angular Aria patterns through experimentation
3. **Document Findings** - Capture insights and patterns as we learn
4. **Demo Ready** - Prepare working examples for meetup presentation

## Angular Aria Focus Areas

- ARIA attributes and roles in Angular components
- Keyboard navigation patterns
- Screen reader compatibility
- Focus management
- Accessible form controls
- Live regions and dynamic content announcements

## Development Approach

- Use standalone components (modern Angular pattern)
- Keep examples minimal and focused
- Test accessibility with keyboard navigation
- Validate ARIA implementation patterns
- Build reusable accessible components

## Project Structure

- `apps/deep-dive-angular-aria` - Main demo application
- `libs/feat-seat-selection` - Feature library for seat selection component (accessibility showcase)

## Key Principles

- Accessibility is not optional - build it in from the start
- Test with keyboard only (no mouse)
- Use semantic HTML before ARIA
- ARIA attributes should enhance, not replace, semantic markup
